# Simul MVP 요구사항 기능서

> **입어보고, 모으고, 공유하다**
> AI 가상시착 결과를 중심으로 개인 옷장을 구성하고 커뮤니티에 공유하는 패션 소셜 플랫폼

---

## 서비스 개요

| 항목 | 내용 |
|------|------|
| 타겟 사용자 | 패션에 관심 많은 20–35세 일반 소비자 |
| 플랫폼 | 반응형 웹 (Web, 모바일 최적화) |
| 개발 범위 | MVP — 핵심 3기능 중심 |

**서비스 설명**
패션에 관심 많은 일반 소비자가 실제 구매 전 AI로 옷을 시각적으로 경험하고, 시착 결과를 커뮤니티에 공유하며, 관심 의류 이미지를 개인 옷장에 모아둘 수 있는 AI 패션 소셜 서비스.

### 기능 우선순위

| 순위 | 기능 | 중요도 |
|------|------|--------|
| 1 | AI 가상시착 (Virtual Try-On) | P1 — 핵심 차별점 |
| 2 | 개인 옷장 (디지털 클로젯) | P2 — 핵심 리텐션 |
| 3 | 커뮤니티 피드 + 시착 결과 공유 | P3 — 성장 엔진 |

## 1. AI 가상시착 `P1`

### 목표
사용자가 자신의 사진에 옷을 AI로 입혀보고 결과를 즉시 비공개 게시글로 보관 및 스와이프 비교, 공유할 수 있다.

### 사용자 플로우

**Step 1 — 사람 이미지(베이스 샷) 입력** (3가지 경로)
- ① 시스템 DB에 이전에 등록해 둔 내 베이스 사진 목록에서 선택 (중복 사용 효율화)
- ② 기기 갤러리 또는 카메라로 새 사람 사진 업로드 시 DB에 자동 보관
- ③ 이전에 이미 AI 시착으로 만들어낸 성공적인 시착 결과를 다시 내 모델 사진으로 지정하여 재사용 순환 등록
- 얼굴 인식 불필요 (신체 실루엣 기반 처리)

**Step 2 — 옷 선택** (3가지 경로)
- ① 내 옷장에 보관된 아이템(이미지) 선택
- ② 기기에서 새 옷 이미지 직접 업로드
- ③ 피드에 올라온 다른 사용자의 옷 정보 복사

**Step 3 — AI 생성**
- 서버 사이드 AI 모델 비동기 처리 및 SSE(Server-Sent Events) 상태 스트림 연결
- 처리 시간 목표: **30초 이내** (초과 시 1회 재시도)
- 대기 중 **랜덤 인기 피드 숏폼 스와이프**: 생성 대기 화면에서 다른 유저의 시착 결과를 세로 풀스크린 스와이프로 탐색하며 좋아요 가능 (기존 피드/좋아요 API 재사용)
- SSE 스트림은 백그라운드 유지하며, 생성 완료 시 "시착 완료! 결과 보기" 오버레이 배너를 즉시 노출

**Step 4 — 결과 확인 및 보관**
- 생성된 시착 이미지는 사용자의 프로필에 **'비공개(is_public=false) 게시물'** 형태로 **자동 영구 보관** (30일 만료 없음)
- 원본 사진 ↔ 시착 결과 스와이프 비교
- 마음에 들 경우 "게시물 만들기" 버튼 탭 → **게시물 생성 화면(SCR-012)으로 이동**, 시착 결과 이미지가 첫 번째로 자동 완성됨
- 게시물 생성 화면에서 추가 이미지 선택 가능 (로컬 앨범 또는 서버에 저장된 다른 시착 결과)
- 시착 시 생성되는 모든 이미지는 DB에 영구 저장 (Redis 캐시 미사용)

### 제약 조건
- 1일 무료 시착 횟수: **5회** / 생성 성공 시 1회 차감, 실패 시 미차감 
- 크레딧 일일 초기화 기준: **매일 KST 00:00:00 (Asia/Seoul)** 기준으로 리셋

## 2. 개인 옷장 (디지털 클로젯) `P2`

### 목표
사용자가 관심 있는 옷 이미지를 중앙 데이터 허브로서 디지털 옷장에 쉽게 단건 보관하고 필터링할 수 있다.

### 사용자 플로우

**Step 1 — 아이템 추가** (4가지 경로)
- ① 사용자가 직접 이미지 업로드 (카테고리 선택사항)
- ② 타 유저 피드나 옷장에서 '내 옷장으로 복사/저장하기' (Deep Copy 로직으로 원본 삭제에 영향 안 받음)
- ③ AI 시착 성공 시 사용했던 의류 이미지는 옷장에도 자동 저장 통합

**Step 2 — 아이템 정보 보완**
- 카테고리는 선택사항 (top, bottom, outer, shoes, accessory)
- 개인 메모 작성 (100자)
- 구글 Vision API 기반 유사 이미지 검색: 사용자가 원할 때 버튼을 탭하면 **쇼핑몰 유사 정보 및 구글 검색 결과**를 DB에 저장하지 않고 일회성으로 노출하여 외부 쇼핑 연결

**Step 3 — 옷장 탐색**
- 카테고리 탭 필터 적용 시 필터링 된 이미지 그리드 표시
- 무거운 브랜드, 쇼핑URL 저장 없이 의류 이미지 위주의 라이트한 갤러리 형태로 표시 

**Step 4 — 아이템 활용**
- 해당 아이템으로 즉시 AI 가상 시착 실행
- 아이템 삭제 (소프트 딜리트 처리)

### 제약 조건
- MVP 아이템 저장 상한: 계정당 **200개**
- 아이템 원본 이미지 크기: 최대 10MB

- ~~코디 조합 저장 (룩북)~~

## 3. 커뮤니티 피드 + 시착 결과 공유 `P3`

### 목표
사용자들이 시착 결과 이미지 단 한 장을 명료하게 피드에 공유하고 소셜 인터랙션으로 소통한다.

### 사용자 플로우

**Step 1 — 게시물 업로드 및 발행** (2가지 경로)
- 시착 직후: 결과 화면에서 "게시물 만들기" 버튼 탭 → 게시물 생성 화면으로 이동, **시착 결과 이미지가 첫 번째로 자동 완성**, 추가 이미지 선택 가능
- 일반 진입: **로컬 앨범** 또는 **서버에 저장된 시착 결과**에서 여러 장의 이미지를 다중 선택하여 업로드 (최소 1장, 최대 5장)
- 캡션 텍스트 (선택, 300자)

**Step 1a — 자동 태그**
- 이미지 업로드 시 Google Vision API로 옷 관련 키워드 자동 추출
- 추출된 키워드를 태그로 자동 추천 (자동완성 형태로 제공)
- 사용자가 취향에 따라 +/- 편집 (최대 10개)
- **태그는 게시물에 항상 노출** — 사용자 부담 최소화

**Step 2 — 피드 탐색**
- 최신/인기순 및 전체/팔로잉 탭 분류 (MVP)
- 이미지 중심 2열 그리드 레이아웃 (무한 스크롤)

**Step 3 — 게시물 상세**
- 다중 이미지 스와이프 조회
- 좋아요 버튼 / 댓글 연동
- 태그 칩 노출 (탭 시 해당 태그 검색 결과로 이동)
- "이 게시물의 옷 구경하기" 버튼으로 원작자의 출처 화면으로 딕링크 연동

**Step 4 — 통합 검색**
- 피드 상단 검색바를 통한 통합 검색 기능
- `#`을 입력하면 태그 검색 모드로 전환
- 검색어를 입력하는 동안 하단에 카테고리별 태그 미리보기 표시
  - 예: `#데님` 입력 → "데님" 태그가 부착된 게시물 미리보기

**Step 5 — 소셜 인터랙션**
- 하트 아이콘 탭하여 낙관적 업데이트 수행
- 최대 2 Depth(대댓글) 포맷의 댓글 커뮤니케이션 추가
- 팔로우/언팔로우 연동

### 제약 조건
- 게시물당 이미지는 최소 1장, 최대 5장
- 이미지 파일 당 최대 20MB 제한
- 게시물당 태그 최대 10개

## 5. 관리자 (Admin / Backoffice) 지원

### 목표
원활한 서비스 운영 및 CS 대응을 위해 백엔드 API 레벨에서 필수 관리 기능(삭제, 보상 등)을 지원한다. (MVP 기간 내 프론트엔드 어드민 페이지 구축은 생략하고 API로 대체)

### 주요 기능
- **콘텐츠 모더레이션**: 신고된 게시물 강제 블라인드 처리 및 악성 댓글 삭제
- **자동 블라인드 (신고 누적 방어)**: 특정 게시물이 단기간에 **N회(기본값: 5회) 이상 신고 접수** 시, 관리자 조치 이전이라도 시스템이 **자동으로 피드에서 임시 블라인드 처리**(`is_blinded=true`)한다. 관리자 확인 후 복구(해제) 또는 영구 삭제를 결정한다.
- **사용자 및 CS 관리**: 불량 사용자 정지(`is_active=false`), 시착 실패 및 오류 발생 시 크레딧 수동 보상(지급)
- **인프라 관제**: AI 서버 부하 모니터링 및 전체 시스템 점검 모드 전환 제어

## 7. MVP 제외 범위 (전체 요약)

| 기능 | 이유 |
|------|------|
| **자체 어드민 웹 페이지** | 개발 공수 삭감을 위해 백엔드 API 및 DB 툴 직접 제어로 대체 운영 |
| **제품 태그 연동(상품 쇼핑)** | 복잡도 극대화, 시착 및 룩 자랑 커뮤니티 본질에 우선 집중하기 위해 전면 제외 |
| **강제 DB 메타 입력(브랜드/색상)** | 데이터 관리 부담 삭감을 위해 구글 Vision API 기반 On-Demand 일회성 검색만 허용 |
| 푸시 알림 고도화 | 기본 알림만 MVP 포함 |
| 어필리에이트 수익화 | 트래픽 규모 확보 후 파트너십 체결 적용 유보 |
| 코디 조합 저장 (룩북) | 옷장 아키텍처 안정화 이후 2차 마일스톤 |
