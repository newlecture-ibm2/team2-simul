# Simul 모바일 뷰포트 레이아웃 규칙

> 모바일 기기(특히 iPhone SE와 같은 최소형 기기)에서 하단 바(Navigation Bar 등)로 인해 주요 콘텐츠가 가려지는 문제를 방지하고, 최적화된 앱 경험을 제공하기 위한 규칙입니다.

## 1. 주요 콘텐츠 권장 가로·세로 크기
- **기준 기기**: iPhone SE(375px) 또는 최소형 기기(320px)를 하한선으로 잡고 설계합니다.
- **가로 너비 (Width)**: 320px ~ 375px. 양옆 최소 16px ~ 20px의 여백(Gutter)을 고려하여 실제 콘텐츠가 들어가는 영역 너비는 280px ~ 340px로 유지합니다.
- **세로 높이 (Height)**: 600px ~ 800px. 상단 상태바 및 하단 내비게이션 바가 차지하는 영역(Safe Area)을 고려하여 핵심 요소(CTA 버튼, 주요 텍스트 등)는 500px ~ 600px 이내에 배치합니다.

## 2. 레이아웃 대응 원칙
1. **상대 단위 사용**: 고정된 `px` 대신 화면 크기에 비례하는 단위(`vw`, `vh`, `dvh`, `%`)를 적극 사용합니다.
2. **dvh (Dynamic Viewport Height)**: 모바일 브라우저 주소창의 유무에 따라 동적으로 변하는 `100dvh`를 기본 컨테이너의 높이 단위로 사용합니다. (이전 `100vh`는 하단 영역이 잘릴 수 있음)
3. **유연한 레이아웃 (Flexbox & CSS Math)**: 남는 공간을 계산할 때 과도한 `padding` 및 고정 `margin`에 의존하기보다는, 컨테이너를 `display: flex`로 설정하고 자식 요소가 `flex: 1`, `margin: auto`, `min()` 등을 활용하여 동적으로 크기를 조절하도록 합니다.

## 3. 전역 비율 원칙
1. **폰트 크기 최적화**: 작은 화면에서 한 번에 더 많은 정보를 보여주기 위해, 전체 폰트 크기(디자인 토큰)를 기존 대비 축소하여 운영합니다. 가독성을 위해 가장 작은 캡션 폰트는 **10px(0.625rem)**로 제한하고 순차적으로 늘어나는 스케일을 사용합니다.
2. **헤더 및 네비게이션 압축**: 콘텐츠 영역을 최대한 확보하기 위해 헤더 및 기타 고정 UI 컴포넌트의 높이를 최소화합니다.

## 대대적 레이아웃 수정 적용 사항 (Modify Plan)

이 규칙을 바탕으로 프로젝트 전반에 다음과 같은 수정 사항이 반영되었습니다. `/community/*` 및 `/closet`의 그리드/폴더 뷰를 제외한 화면이 일반적인 모바일 뷰포트 내에 들어오도록 최적화합니다.

### 1. 전역 디자인 토큰 수정 (폰트)
- `--font-display`: `clamp(1.6rem, 4vw + 0.8rem, 3.5rem)`
- `--font-heading`: `clamp(1rem, 1.6vw + 0.6rem, 1.4rem)`
- `--font-body`: `clamp(0.75rem, 0.8vw + 0.5rem, 0.875rem)` (12px ~ 14px)
- `--font-caption-bold`: `0.75rem` (12px)
- `--font-caption`: `0.625rem` (10px)
- `--font-button`: `0.625rem` (10px)

### 2. 헤더 높이 축소
- `.headerInner`: `padding: 10px var(--space-page-x)`
- `.logoImage`: `height: 15px`
- `.iconBtn`: `width: 32px`, `height: 32px`
- `.iconImage`: `width: 20px`, `height: 20px`

### 3. 홈 화면 (SwipeDeck) 최적화
- 부모 컨테이너(`deckContainer`)를 `flex`로 구성하고, 전체 높이를 헤더를 제외한 뷰포트 높이(`calc(100dvh - 52px)`)로 설정.
- 카드 컨테이너(`deckWrapper`)가 `min(100%, calc((100dvh - 250px) * 4 / 6))` 너비를 갖도록 설정하여 화면 높이가 짧아져도 비율을 유지한 채 동적으로 크기가 줄어들도록 개선. 고정된 마진 및 패딩 제거.

### 4. 가상 시착 (TryOn) 홈 압축
- 상단 히어로 영역 패딩 축소 (`padding: 24px 16px`), 내부 요소 간격 축소 (`gap: 16px`).
- 대표 이미지 카드의 고정 높이를 `240px`에서 `180px`로 축소하여 600px 뷰포트 내에서도 "시착 시작하기" 버튼이 한눈에 보이도록 압축.

### 5. 프로필 (Profile) 상단 영역 축소
- 상단 히어로 영역의 최소 높이 하한선을 `min-height: 340px`에서 `min-height: 250px`로 축소.
- 작은 화면에서도 하단의 게시물/옷장 탭이 최대한 보이도록 조정.
