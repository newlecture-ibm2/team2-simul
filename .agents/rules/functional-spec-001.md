## AI 가상시착 핵심 규칙

### FN-103 AI 이미지 생성 처리 흐름
1. 크레딧 잔여 확인 (0이면 ERR-103-A)
2. AI 생성 API 비동기 호출 (job_id 반환)
3. SSE 스트림으로 실시간 상태 전송
4. 대기 중 랜덤 인기 피드 숏폼 스와이프 노출
5. 완료 시 **비공개 게시물(is_public=false)** 자동 생성→SCR-024
6. **성공 시에만** 크레딧 1 차감
7. 실패/타임아웃 시 자동 재시도 1회, 크레딧 미차감

### 베이스 이미지 제약
- 포맷: JPG/PNG/HEIC | 최소 256×256 | 최대 20MB
- 시착 결과를 새 베이스로 복제 등록 가능 (순환 구조)

### 크레딧 시스템
- 일일 5회 무료 | KST 자정 리셋 | 관리자 수동 지급 가능

### 시착 결과
- 모든 시착 이미지 DB 영구 저장 (비공개 게시물)
- SCR-024 "게시물 만들기"→SCR-012 이동 시 시착 결과 첫 이미지 자동 완성

## 개인 옷장 핵심 규칙

### 아이템 추가 4가지 경로
1. **직접 추가** (SCR-032) — 이미지 업로드+카테고리+메모
2. **타인 옷장에서 복사** (SCR-031) — Deep Copy (독립 객체)
3. **시착 시 자동 저장** (FN-103) — 시착 완료 시 옷장 자동 보관
4. **피드 태그에서 저장** (SCR-011) — 태그된 옷 이미지 복사

### 제약
- 계정당 최대 **200개** (ERR-201-A)
- 이미지 최대 10MB (ERR-201-B)
- 삭제는 소프트 딜리트 (`deleted_at`)
- 타인이 복사한 사본은 원본 삭제 시 영향 없음

### 유사 이미지 검색 (FN-204) [Should]
- Google Vision API Web Detection → 일회성 결과 표출 (DB 미저장)
